# Data, privacy, and sharing boundaries

## Paper 1

The anonymized survey dataset may be shared only under the conditions described in the manuscript data-availability statement. Direct identifiers, encrypted participant identifiers, contact details, and free-text content that could identify respondents must not be committed to GitHub.

## Paper 2

Authentic student assignments are subject to institutional, ethical, and privacy restrictions. GitHub should contain code, manifests, schemas, aggregate outputs, and synthetic examples—not the private assignment corpus. A controlled repository or access-on-request mechanism should be used when required by the journal.

## Paper 3

The controlled benchmark should be deposited separately. Public-source AI-Free artifacts must be checked for redistribution rights; when redistribution is uncertain, publish source metadata/URLs rather than republishing copyrighted content. AI-generated and humanized files, archived detector outputs, manifests, and analysis datasets can be deposited when permitted.

## Across all papers

Never commit API keys, raw personal data, student names/numbers, instructor-identifying metadata, unpublished reviewer comments, or private repository URLs.
